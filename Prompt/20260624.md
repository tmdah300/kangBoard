### APIエンドポイント一覧

| メソッド | パス | 認証 | 説明 |
|---|---|---|---|
| POST | /api/auth/register | 不要 | 新規登録 |
| POST | /api/auth/login | 不要 | ログイン |
| GET | /api/posts | 不要 | 投稿一覧（DelFlag=falseのみ） |
| GET | /api/posts/{id} | 不要 | 投稿詳細（閲覧数+1） |
| POST | /api/posts | 必要 | 投稿作成 |
| DELETE | /api/posts/{id} | 必要 | 投稿削除（DelFlag=true） |
| GET | /api/posts/{id}/comments | 不要 | コメント一覧 |
| POST | /api/posts/{id}/comments | 不要 | コメント投稿 |
| DELETE | /api/posts/{id}/comments/{commentId} | 必要 | コメント削除（DelFlag=true） |
| POST | /api/posts/{id}/like | 必要 | いいね+1 |

### フロントエンド画面と権限

| 画面 | パス | 未ログイン | ログイン済み |
|---|---|---|---|
| 投稿一覧 | / | 閲覧のみ・案内文表示 | 削除ボタン表示 |
| 投稿詳細 | /posts/:id | 閲覧のみ・いいねはアラート | 削除・いいね可能 |
| 新規投稿 | /posts/new | ボタン押下でアラート表示 | 投稿可能 |
| ログイン | /login | - | - |
| 新規登録 | /register | - | - |

### 起動方法

**VS Codeでフォルダを開く（推奨）**
```
VS Code で Board_geachu フォルダを開く
→ 「Allow」を押す（初回のみ）
→ APIサーバー・フロントエンドが自動起動
```

**手動起動**
```bash
# API（PowerShell or ターミナル）
cd BoardApi
dotnet run

# フロントエンド（Git Bash）
cd board-front
npm run dev
```

### 既知の問題・注意事項

- APIサーバーは `dotnet run` で起動したウィンドウを閉じると落ちる。VS Code のタスク自動起動を使うこと。
- 削除は論理削除（DelFlag=true）のみ。物理削除の機能はない。
- 投稿・コメントの所有者管理（誰が書いたか）は未実装。現状はログイン済みであれば誰でも全投稿・コメントを削除できる。
- JWT はローカルストレージに保存。本番環境では HttpOnly Cookie 推奨。
